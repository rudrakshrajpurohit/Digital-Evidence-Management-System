# AI-Based Earthquake Risk & Alert Dashboard

## 1. Overview

This project presents a simple yet practical approach to disaster management using artificial intelligence. The system is designed as a web-based dashboard that provides real-time earthquake data, analyzes potential risk levels, and visually communicates this information through an interactive map interface.

The goal is not to predict earthquakes with high complexity, but to create an accessible and responsive system that helps users quickly understand the severity of an event and take informed action.

---

## 2. Problem Statement

Earthquakes are sudden and unpredictable natural disasters that often lead to significant loss of life and infrastructure damage. While seismic monitoring systems exist, the information is not always easily accessible or understandable to the general public.

Most people lack:

* Real-time awareness of nearby seismic activity
* Clear understanding of how dangerous a situation is
* Simple tools to interpret technical earthquake data

This gap between data availability and usability reduces the effectiveness of early response.

---

## 3. Why This Problem Matters

The impact of delayed or unclear information during disasters can be severe:

* People are unable to react quickly due to lack of awareness
* Emergency response systems face delays in decision-making
* Public panic increases due to misinformation or confusion

A system that simplifies and visualizes this information can significantly improve response time and awareness.

---

## 4. Proposed Solution

The proposed solution is an AI-powered dashboard that collects real-time earthquake data and presents it in an intuitive and user-friendly manner.

The system:

* Fetches live earthquake data from reliable sources
* Processes key parameters such as magnitude and depth
* Uses a simple AI model to classify risk levels
* Displays results on an interactive map with clear visual indicators

This approach ensures that even non-technical users can quickly understand the situation.

---

## 5. Key Features

* Real-time earthquake data integration
* Risk classification into Low, Medium, and High categories
* Interactive map visualization of affected regions
* Simple and clear alert representation
* Lightweight and fast system design

---

## 6. How the System Works

The system follows a straightforward pipeline:

1. Earthquake data is fetched from an external API (e.g., USGS)
2. The backend processes this data
3. An AI model evaluates risk based on selected parameters
4. The processed data is sent to the frontend
5. The frontend displays results on a map with visual indicators

This flow ensures efficient data handling and near real-time updates.

---

## 7. AI Concept Used

The project uses a basic machine learning classification approach.

* Input features: magnitude, depth
* Output: risk level (Low / Medium / High)

Initially, a rule-based or simple classification model is used as a baseline. This keeps the system lightweight while still demonstrating the application of AI in disaster management.

The design allows future integration of more advanced models such as time-series prediction or deep learning.

---

## 8. Tech Stack

Frontend:

* React for building the user interface
* Leaflet.js for map visualization

Backend:

* FastAPI for API development and data handling

AI/ML:

* Python
* Scikit-learn for basic classification

Data Source:

* USGS Earthquake API for real-time data
